I'm writing this post about the Chaos Tools for the presentation I am making at the Drupal Dev Days on May 8th in München.

What does provide by ctools?

Useful, well written and easy to use scripts / API s for Drupal module development.

The widgets and interactivity tools

  • AJAX responder
  • Dependent
  • Modal dialog
  • Wizards
  • Jump menu

AJAX responder

tools to make it easier for the server to handle AJAX requests and tell the client what to do with them.

In other words, this the perfect "inline editing" solution.

Imagine you can click an HTML element, the AJAX responder could determine the appropriate form element and put the form at a defined position within the page.
After submiting the change, the original element would be replaced. All that without leaving the current page.

Ok, this is generaly speeking one of the most standard use case. Chaos Tools just make it easier.

[ctools-demos:ajax]

The APIs

  • Forms utilities
  • Exportable objects
  • Object cache
  • Context
  • CSS tools

Plugins

tools to make it easy for modules to let other modules implement plugins from .inc files.

You might also implement some plugins from themes, like the panels layouts and styles (this is cleary a good choice to me because it really allows to separate the information from the design - like CSS & XHTML).

The plugin definition can also be an ".info" file.

[ctools-demos:plugins]

Exportables

tools to make it easier for modules to have objects that live in database or live in code, such as 'default views'.

tools to make it easier for forms to deal with AJAX.

Object caching

tool to make it easier to edit an object across multiple page requests and cache the editing work.

Even better, this API provides methods to lock an editing state to prevent other users editing conflicts.

[ctools-demos:object-cache]

Modal dialog

tool to make it simple to put a form in a modal dialog.

[ctools-demos:modal]

Dependent

a simple form widget to make form items appear and disappear based upon the selections in another item.

[ctools-demos:dependent]

Content

pluggable content types used as panes in Panels and other modules like Dashboard.

Form wizard

an API to make multi-step forms much easier.

[ctools-demos:wizards]

CSS tools

tools to cache and sanitize CSS easily to make user-input CSS safe.

[ctools-demos:css]

Zusammenfassung I ♥ that word...

CTools is the perfect library/framework to create advanced editing forms/UIs.

It is also very helpful for module aimed to edit and share complex configurations.

The context approach is really relevant/flexible when dealing with complex site workflows/structures.

The plugins system is an amazing way to make the extent the possibilities of a module.
